POST /trips/:tripId/unlock Avvia pagamento unlock match JWT ▼
Request body
{
  "matchId": "match_xyz",
  "fakeDoor": false          // true in VITE_FAKE_DOOR_MODE=true
}
Response 200 — live mode
{
  "paymentIntentClientSecret": "pi_xxx_secret_yyy",
  "amount": 99,               // cents, da airport.unlockFee
  "currency": "eur"           // da airport.currency
}
Response 200 — fake door mode
{
  "fakeDoor": true,
  "message": "Intent recorded"
}
Flusso live: PaymentIntent con capture_method: manual. Capture avviene solo quando entrambi gli utenti hanno autorizzato. Se solo uno paga entro 24h → void. Fake door: nessun addebito reale, emette payment.simulated su EventBridge.

POST /webhooks/stripe Webhook Stripe (no auth) Stripe-Sig ▼
Header richiesto
Stripe-Signature: t=...,v1=...
Eventi gestiti
payment_intent.amount_capturable_updated → capture se entrambi autorizzati
payment_intent.payment_failed             → void + notifica
identity.verification_session.verified    → aggiorna verified=true
customer.subscription.updated             → aggiorna isPro
customer.subscription.deleted             → aggiorna isPro=false
Response 200
{ "received": true }
Idempotency: ogni evento verificato via Stripe-Signature e deduplicato tramite eventId in DynamoDB.
WS $connect Apri connessione ▼
URL
wss://ws.flot.app/{stage}?token=<jwt>
Lambda valida JWT → salva CONN#{connId} in DynamoDB (TTL 24h). Il frontend apre la WS al mount di /search.

BUS flot-events EventBridge custom bus ▼
Eventi pubblicati e loro consumer
user.created          → Welcome email (SES) + analytics
trip.created          → Matching engine (search trips GSI1)
match.found           → WebSocket push + push notification (SNS)
payment.completed     → Unlock match + abilita chat
payment.simulated     → Log fake door intent (FAKE_DOOR_MODE)
payment.voided        → Notifica utente pagamento fallito
trip.completed        → Richiedi review + schedule chat cleanup (48h TTL)
subscription.active   → Aggiorna isPro=true su utente
subscription.ended    → Aggiorna isPro=false su utente
Schema evento standard: 
{
  "source": "flot.backend",
  "detail-type": "match.found",
  "detail": {
    "matchId": "match_xyz",
    "airportCode": "MXP",
    "userId1": "user_123",
    "userId2": "user_456"
  }
}
